Local Meet Translator

Bridge runs on localhost. The extension uses a local token.

Use 4–6 for stability.

Outgoing voice (my mic → translated voice → meeting)

To actually send the translated voice to the other participant, you must route the TTS into a virtual cable output device and select the matching virtual microphone in the meeting app.
If device names are blank, grant mic access once (button below), then reopen the popup.
5 is a good default.
This will prompt for microphone permission and refresh the device list.
Select the virtual cable playback device (e.g. “CABLE Input”). If left as default speakers, the other side will not hear the translated voice.
Recommended OFF. When ON, the overlay will also show what the extension thinks you said.
