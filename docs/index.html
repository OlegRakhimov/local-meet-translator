How it works

Extension captures audio

Captures call audio from the active tab for subtitles, and your physical microphone for outgoing translation.

Local bridge talks to AI

A small local service on localhost handles transcription, translation, and optional TTS. Your API key stays on your machine.

Clean routing

You see translated subtitles. The other participant hears translated voice via a virtual microphone. Subtitles do not include your own voice by default.

Setup in 3 minutes

1

Install the extension

Install from Chrome Web Store / Edge Add-ons and open the popup.

2

Start the local bridge

Run the provided script and confirm /health is OK in the extension.

3

Choose your mode

  • Subtitles-only: no virtual audio required.
  • Voice to the other participant: route TTS to a virtual mic (VB-Cable).

Voice mode routing (recommended)

  1. Extension mic input: your real microphone
  2. Extension TTS sink: VB-Cable “CABLE Input”
  3. Google Meet microphone: VB-Cable “CABLE Output”
  4. Meet speakers: your real headphones/speakers

This keeps your own TTS out of your mic and prevents “talking to itself”.

Privacy & data

API key stays local

The extension talks only to your localhost bridge. Store your AI credentials in a local .env file.

Consent matters

If you use call recording/transcription features, ensure you have the necessary consent from participants and comply with local regulations.

FAQ

Does this work without a virtual audio cable?

Yes. Subtitles-only mode works without any virtual devices. Voice mode requires routing TTS into the call via a virtual microphone.

Why do I sometimes see “random phrases”?

Most ASR systems can hallucinate on silence. This project drops low-energy chunks and pauses mic capture during TTS playback to reduce self-feedback.

Which platforms are supported?

Any web call that runs in the browser can work. Google Meet is the primary target; Zoom/Teams web may vary by permissions and audio routing.

How do I get support?

Provide diagnostics logs and your audio device routing (mic input, TTS sink, call microphone).
